# Fighter AI — Concept A
## LSTM Move Predictor + CNN Counter-Action Classifier

A 2D fighting game where a human plays against an AI opponent powered by
two chained neural networks. The LSTM watches your move history and predicts
your next action; the CNN uses that prediction to pick the optimal counter.

## Quick start (recommended)

### 1. Install dependencies

```bash
# From inside the fighter_ai/ directory
pip install -r requirements.txt
```

PyTorch CPU install (if no GPU):
```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install numpy pygame scikit-learn matplotlib
```

PyTorch GPU install (CUDA 12.1):
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
pip install numpy pygame scikit-learn matplotlib
```

### 2. Run the full pipeline with one command

```bash
bash run.sh
```

This runs all four steps in order:
1. Collect training data (bot matches)
2. Train the LSTM (Model 1)
3. Train the CNN (Model 2)
4. Run the ablation study
5. Launch the game

Or run each step individually:

```bash
bash run.sh collect    # Step 1: generate data (~2 min, 300 matches)
bash run.sh train      # Step 2: train both models (~10–20 min CPU)
bash run.sh ablation   # Step 3: HD evaluation (100 rounds each)
bash run.sh play       # Step 4: play the game
```

## Step-by-step manual commands

All commands must be run from the `fighter_ai/` directory.

### Step 1 — Generate training data

```bash
python training/collect_data.py --matches 300 --seed 42
```

**What it does:** Runs scripted bots (aggressive, defensive, combo, random)
against each other for 300 headless matches and saves the game state at
every frame.

**Output files in `data/`:**
- `raw_frames.npy`       — shape `(N, 13)` normalised feature vectors
- `p1_actions.npy`       — shape `(N,)` P1 action label each frame
- `p2_actions.npy`       — shape `(N,)` P2 action label each frame
- `match_boundaries.npy` — frame indices where each match starts

**Options:**
```bash
--matches 500    # more data = better models (500 takes ~3–4 min)
--seed 99        # different random seed
--out my_data    # custom output directory
```

---

### Step 2a — Train the LSTM (Model 1)

```bash
python training/train_lstm.py --epochs 30 --batch 256 --lr 1e-3
```

**What it does:**
- Loads the frame data and builds sliding windows of length 10
- Trains a bidirectional 2-layer LSTM to predict the human's next move
- Uses class-weighted cross-entropy loss to handle action imbalance
- Saves the best checkpoint (by val F1) to `checkpoints/lstm_best.pt`
- At the end, pre-computes LSTM probability outputs for all frames
  and saves them to `data/lstm_probs_cache.npy` (needed for CNN training)

**Output:**
- `checkpoints/lstm_best.pt`
- `logs/lstm_train_log.csv`
- `plots/lstm_confusion.png`
- `plots/lstm_roc.png`
- `data/lstm_probs_cache.npy`

**Options:**
```bash
--epochs 50    # more epochs if val F1 is still improving
--batch 128    # reduce if you get OOM errors
--lr 5e-4      # lower LR for more stable training
```

**What to look for:**
- Val F1 should climb above 0.35 by epoch 10
- If it's stuck below 0.20, increase `--matches` in data collection
- The confusion matrix will show which moves get confused most

---

### Step 2b — Train the CNN (Model 2)

```bash
python training/train_cnn.py --epochs 25 --batch 512 --lr 5e-4
```

**What it does:**
- Loads game state features + pre-computed LSTM probability outputs
- Trains a 1D CNN to classify the best counter-action for the AI fighter
- Uses OneCycleLR scheduler for faster convergence
- Saves the best checkpoint to `checkpoints/cnn_best.pt`

**Must run after train_lstm.py** (requires `data/lstm_probs_cache.npy`).

**Output:**
- `checkpoints/cnn_best.pt`
- `logs/cnn_train_log.csv`
- `plots/cnn_confusion.png`

---

### Step 3 — Ablation study (HD evaluation)

```bash
python training/ablation.py --rounds 100
```

**What it does:**
Runs 100 complete rounds of the AI against three opponent types (random,
aggressive, defensive bot), twice — once with just the CNN, once with the
full CNN+LSTM pipeline. Reports win rate, average health remaining, and
inference latency per frame.

This is your key HD evidence: if CNN+LSTM beats CNN-alone, the integration
is empirically justified.

**Output:** `logs/ablation_results.csv` + printed summary table.

Example output:
```
Opponent           Model          Win %   HP left   ms/frame
──────────────────────────────────────────────────────────────
Random agent       CNN alone      61.0%      42.3       0.82
Random agent       CNN + LSTM     74.0%      51.7       1.14
Aggressive bot     CNN alone      48.0%      28.1       0.83
Aggressive bot     CNN + LSTM     59.0%      35.4       1.15
Defensive bot      CNN alone      55.0%      38.9       0.81
Defensive bot      CNN + LSTM     67.0%      44.2       1.13
```

### Step 4 — Play the game

```bash
python game/play.py
```

**Controls:**

| Key     | Action     |
|---------|------------|
| J       | Punch      |
| K       | Kick       |
| L       | Block      |
| Space   | Jump       |
| S       | Crouch     |
| U       | Fireball   |
| I       | Uppercut   |
| +       | Harder AI  |
| -       | Easier AI  |
| R       | Restart (after round ends) |

**HUD overlay (bottom-left):**
Shows the LSTM's top-3 predicted moves with probability bars in real time.
This is the live visualisation of Model 1 working — great for the video demo.

**Difficulty:**
The `--temperature` argument (or +/- keys in-game) controls the CNN's
softmax sharpness. Lower = smarter AI, higher = more random/forgiving.

```bash
python game/play.py --temperature 0.3   # challenging
python game/play.py --temperature 1.5   # relaxed
```

| Model     | Metric          | What it measures                        |
|-----------|-----------------|------------------------------------------|
| LSTM      | F1 macro        | Equal-weight accuracy across all 8 moves |
| LSTM      | AUC per class   | ROC curve area for each move class       |
| LSTM      | Confusion matrix| Which moves get confused                 |
| CNN       | F1 weighted     | Counter-action accuracy (action-weighted)|
| System    | Win rate        | End-to-end performance vs 3 opponents    |
| System    | Avg HP left     | Dominance margin per win                 |
| System    | Latency (ms)    | Inference speed (must be < 16ms for 60fps)|
| Ablation  | Δ win rate      | CNN+LSTM improvement over CNN alone      |

## Troubleshooting

**`pygame.error: No video mode set` (headless server)**
```bash
export SDL_VIDEODRIVER=offscreen
```
Or use `render_mode=None` for training — the game requires a display.

**`ModuleNotFoundError`**
Make sure you're running commands from the `fighter_ai/` directory, not a
subdirectory.

**LSTM val F1 stuck below 0.15**
Increase data: `--matches 500`. Also check class distribution printed
at the end of collect_data.py — if IDLE > 60%, the weighting is working
but you may need more aggressive bots.

**OOM (out of memory)**
Reduce `--batch` to 64 or 128.

**Game is too fast/slow**
FPS is set to 60 in `env/fighter_env.py` (constant `FPS`). Lower it for
slower machines.
